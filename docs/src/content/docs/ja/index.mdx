PosterPilot は、メディアライブラリ
（[Plex](https://www.plex.tv/)、暫定的な [Jellyfin](https://jellyfin.org/)
および [Emby](https://emby.media/) サポート付き）を閲覧し、複数のプロバイダーをまたいでアートワークのカバーを
見つけ、選んだカバーをメディアサーバーに適用するセルフホスト型ウェブアプリです — その API を通じて直接、
かつ／または [Kometa](https://kometa.wiki/)/PMM 互換の YAML をエクスポートすることで適用できます。これは
Mac または Unraid サーバー上で単一の Docker コンテナとして動作します。

## 機能

<CardGrid stagger>
	<Card title="ライブラリを同期" icon="seti:db">
		映画と番組のライブラリを同期し、各タイトルを TMDB id に解決することで、アートワークプロバイダーに
		正確に問い合わせできるようにします。
	</Card>
	<Card title="カバーを見つける" icon="seti:image">
		有効なプロバイダー — MediUX、Fanart.tv、TMDB、ThePosterDB — をまたいで展開し、ポスターと背景の
		候補をプロバイダーとセットごとにグループ化して比較します。
	</Card>
	<Card title="カバーを適用" icon="approve-check">
		選んだカバーを、メディアサーバーの API 経由（フィールドをアップロードしてロックし、即座に反映）で、
		かつ／または、次回の実行時に Kometa インスタンスが取り込む Kometa YAML エクスポート経由で適用します。
	</Card>
	<Card title="ローカライズを維持" icon="translate">
		UI は英語、スペイン語、簡体字中国語、日本語、ブラジルポルトガル語で提供され、キーごとに英語へ
		フォールバックします — Weblate を通じて管理されています。
	</Card>
</CardGrid>

ライブラリ全体にわたる処理は、Server-Sent Events によるライブ進捗付きのバックグラウンドジョブとして
実行されます。

## 技術スタック

- **Bun** 上の **SvelteKit**（TypeScript）、`adapter-node` でビルド
- ライブラリキャッシュ、候補、履歴、ジョブ、設定のための **SQLite + Drizzle ORM**（libsql）
- **Tailwind CSS v4**、ダークで画像中心の UI
- ライブ進捗のための **Server-Sent Events** を備えたインプロセスジョブキュー
