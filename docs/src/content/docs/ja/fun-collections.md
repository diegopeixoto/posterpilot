FUN とコレクションは同期済みライブラリを再利用します。結果やビジュアルファミリーが見つかっただけで、メディア ID を推測したりアートワークを適用したりしません。

## FUN を有効にする

`FUN_ENABLED=true` または設定の **FUN** を有効にします。無効時はナビゲーションが消え、`/fun` は Not Found です。フィルターはアクティブな名前付きサーバーに限定され、抽選前に正確な対象件数を表示します。無効な範囲を勝手に広げません。

## ナイトピッカー

対象から最大 3 件の異なる候補を返します。ライブラリ、種類、ジャンル、年、視聴済み、上映時間、最低評価、追加時期、プリセットで絞り込めます。URL にシード、正規化フィルター、ID を保存するため、項目が存在する限り更新、戻る、共有で同じ順序を再現します。

再抽選はフィルターを保ち新しいシードを使います。候補が十分ならセッションの最近履歴を避け、少ない場合は繰り返し回避だけを緩めます。ブラインドは公開まで ID を隠し、カプセルは抽選前に条件を表示します。

## Poster Match

2 件以上のポスター候補があるタイトルで、2 枚ずつ選び有限のトーナメントを進めます。読み込めない候補は可能な限り除外します。勝者はプロバイダー情報を保ったまま**ステージ**されるだけで、通常のプレビューと確認が必要です。

## アンビエントギャラリー

ポスター、背景、混在を全画面表示し、前後、停止／再生、間隔、終了をキーボードでも操作できます。`prefers-reduced-motion: reduce` では明示的に再生するまで停止状態です。読み込み失敗画像はセッション中スキップします。

## セッションプランナー

2 または 3 本と時間予算を選びます。上映時間が判明した異なる映画だけを使い、ライブラリ、ジャンル、視聴、評価を守り、予算を超えません。再計画は条件を保ってシードを変えます。

## コレクションとフランチャイズ

コレクションはサーバーごとに分離され、ネイティブ所属または TMDB の `belongs_to_collection` から作成されます。別サーバーの同名コレクションは混在せず、タイトル類似だけでは所属させません。

詳細には由来、参考用の未所持 TMDB メンバー、現在／ステージ済み画像、プロバイダー／セット／作者／言語／デザインファミリーの証拠、説明可能な整合率を表示します。由来不明を意図的な不一致とは扱いません。

## 統一候補と上書き

複数メンバーに検証可能な証拠があるファミリーを、カバレッジとスコアで順位付けします。対象／未対象メンバーとスロットを示し、対象だけをステージします。各メンバー／スロットを個別に上書き・解除できます。共通証拠がなければ、統一セットだと主張せず個別候補を表示します。

## コレクションの適用と復元

コレクションでの選択も通常のステージです。統一書き込みには、メンバー、スロット、宛先、選択、現在状態、スキップを凍結した正確なプレビューが必要です。所属や選択の変更で無効になります。結果はメンバー／宛先ごとに残ります。

グループ復元には対応するリビジョンと新しいプレビューが必要です。個別メンバーは項目履歴から操作できます。現在の UI に統一操作がない機能はメンバーを開き、通常の Review／適用／復元を使ってください。ステージだけでは書き込みません。
