PosterPilot の日常処理は永続化され、レビュー優先です。同期、検索、再試行、確認済み適用は保存されたジョブとして動き、自動化がアートワークを自動適用することはありません。

## 永続ジョブ

ダッシュボードには待機、実行、再試行待ち、終了したジョブとライブ進捗が表示されます。再読み込みや画面移動でキャンセルされません。同じ要求は実行中ジョブを再利用し、競合する変更は競合ジョブを示して拒否します。

ジョブはサーバー／ライブラリ範囲、変更されない入力、試行、結果、秘匿化された失敗を保存します。再起動後は安全な作業を再開します。途中で中断した非冪等な書き込みは盲目的に再実行せず、レビュー対象になります。

**キャンセル**は停止を要求しますが、確定済み結果は消しません。部分失敗の**失敗のみ再試行**は対象単位だけを再実行します。検証、認証、古いプランは修正と新しいプレビューが必要です。

## レビュー優先の自動化

**設定 → 自動化**で、アクティブサーバーに対して次を設定します。

- 1 つ以上のライブラリ
- 間隔、毎日の時刻、またはイベント（新規項目／同期完了）
- IANA タイムゾーン
- **同期**または**同期して検索**
- 任意の保存済みレビュービュー
- キャッチアップ時間と連続失敗による一時停止のしきい値

既定は `sync_discover` です。各実行は入力を凍結して 1 つの永続ジョブを作成・再利用します。編集は将来の実行だけに影響します。キャッチアップ時間内の再起動では、未実行分を 1 回だけ作成し、重複イベントはまとめます。

:::important
自動化は同期と任意の候補検索を行い、Review に残します。適用ジョブは作成しません。
:::

### Webhook

自動化画面で資格情報を生成します。エンドポイントとトークンは 1 回だけ表示されます。`X-PosterPilot-Webhook-Token` ヘッダーで送信してください。ローテーションは以前のトークンを無効化し、無効化操作は削除します。URL やログに含めないでください。

## 再試行前の診断

**設定 → 診断**では、各サーバー、TMDB、プロバイダー、Kometa パス、データ、バックアップを個別に、変更せず検査します。到達不能、認証情報不足／拒否、タイムアウト、読み書き権限を区別し、インスタンスごとのアートワーク機能も表示します。

最新結果と最終成功は再起動後も残ります。プロバイダー障害時は既知候補を古い状態として保持でき、その後の正常な空応答で削除します。

秘匿化されたサポートバンドルは明示操作でのみ出力します。タイトルは既定で含まず、安全性を確認できない任意エントリは省略してマニフェストに記録します。

## アプリケーションバックアップ

**設定 → バックアップと復元**から作成します。PosterPilot は SQLite の整合したスナップショットと、チェックサム、アプリ／スキーマ版、キーモード、外部パス参照を持つバンドルをデータディレクトリに保存します。メディアサーバーや外部 Kometa 内容はコピーしません。

`.app-key` モードでは生成キーを含みます。`APP_SECRET` モードではシークレットを含めず、復元時に同じ値が必要です。

検証、エクスポート、削除が可能です。エクスポートは機密情報を含む可能性があるため再確認します。個数／日数の保持設定は、保護されていない有効バンドルだけを削除します。

## 復元

1. **復元をプレビュー**します。
2. チェックサム、SQLite、スキーマ／移行、容量、キー、外部警告を確認します。
3. 変更されていないプランと置換範囲を確認します。
4. メンテナンスモードで新規変更を止め、実行中ジョブを終了させ、保護された安全バックアップを作成します。
5. コンテナを再起動します。libsql が開く前に置換します。
6. 準備状況レポートを確認します。置換や移行失敗時は安全バックアップへロールバックします。

外部サービス停止はローカル整合性があれば警告です。チェックサム、DB、新しすぎるスキーマ、パス、キーの不一致はブロックします。

:::caution
稼働中の SQLite を手動で置換しないでください。ライブラリ、認証情報、Kometa、自動化、サーバー範囲を確認するまで安全バックアップを保持します。
:::
