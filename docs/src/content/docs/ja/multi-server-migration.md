PosterPilot は複数の Plex、Jellyfin、Emby を管理し、ライブラリ、項目、ジョブ、リビジョン、コレクション、レビュー、自動化を厳密に分離します。

## 既存環境の更新前

1. 変更ジョブの完了を待ちます。
2. DB／WAL、スナップショット、バックアップ、`.app-key` を含む `/data` 全体をコピーします。利用できる場合はアプリの手動バックアップも作成・検証します。
3. 現在の種類、URL、認証情報の設定元を記録し、`APP_SECRET` または `.app-key` を保持します。
4. イメージを更新して通常どおり再起動します。空 DB や手動 SQL 移行は作らないでください。

## 移行内容

保護された **Default server** を 1 つ作り、既存レコードを安定した `legacy-default` 範囲へ割り当てます。起動時に有効な従来接続（環境変数が優先）をそのインスタンスへ設定し、アクティブにします。

移行はトランザクションかつ冪等です。ID、候補、選択、無視、ジョブ、履歴、キャッシュ済みライブラリは維持され、破壊的な再同期は不要です。未設定の新規環境では偽の接続を作らずセットアップへ進みます。

:::note
`SERVER_TYPE`、`PLEX_*`、`JELLYFIN_*`、`EMBY_*` は保護された既定接続を表します。追加サーバーは設定画面で作り、個別の暗号化資格情報を保存します。環境変数で任意のサーバー一覧は定義しません。
:::

## 更新後の確認

1. **設定 → サーバー**で種類、URL、資格情報、アクティブ表示を確認します。
2. 接続テストと**診断**を実行します。
3. ライブラリ、Review、コレクション、ジョブ、項目タイムラインの件数を確認します。
4. 対象ライブラリと Kometa の Plex バインドを確認します。
5. 増分同期を実行します。全再スキャンは意図的に全項目を読み直す場合だけ使います。

移行／復号に失敗したら新コンテナを停止し、以前のボリュームまたは検証済み復元フローを使います。部分コピーの DB で運用しないでください。

## サーバーの追加と切り替え

**設定 → サーバー**で一意な名前、種類、URL、再利用可能な資格情報を入力し、テストして追加します。Plex はトークン、Jellyfin／Emby は API キーまたはアクセストークンです。保存済みシークレットはブラウザーへ返しません。

2 台以上が有効ならスイッチャーまたは **Make active** を使います。ページ、フィルター、Review ビューは選択範囲に留まります。ジョブ／自動化も名前付きサーバーを保持し、独立作業は並行可能です。機能はインスタンスごとに異なります。

## Kometa のバインド

Kometa は Plex 専用です。`KOMETA_SERVER_INSTANCE_ID` または設定画面で名前付き Plex を選びます。プレビューでバインドを検証し、Jellyfin／Emby や別 Plex の資格情報を暗黙に借りることは拒否します。

## サーバー間適用

必ず明示操作で、共有する正確な TMDB、IMDb、TVDB ID が必要です。似たタイトルだけでは対象にしません。プレビューはサーバー／項目、機能、スロット、現在状態、選択、スキップを表示し、各宛先に独立したリビジョンと検証を作ります。

通常の適用は伝播しません。UI に選択画面がない場合、プレビュー／確認 API は管理された連携向けです。プレビュー後にアクティブサーバーを切り替えて模倣しないでください。

## 無効化、切断、パージ

- **無効化**は新規変更を止め、資格情報、キャッシュ、履歴を残します。
- **切断**は資格情報を削除し、自動化を無効にして履歴を残します。
- **パージ**は切断後に影響件数を表示し、別の確認を要求します。実行中変更ジョブがあるとブロックされます。

移行された既定サーバーは **Legacy** と表示され、通常の編集／パージから保護されます。パージ前にバックアップを作り、項目、ジョブ、リビジョン、コレクション、自動化、スナップショットを確認してください。
