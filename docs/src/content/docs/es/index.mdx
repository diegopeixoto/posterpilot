PosterPilot es una aplicación web autoalojada que explora una biblioteca
multimedia ([Plex](https://www.plex.tv/), [Jellyfin](https://jellyfin.org/) o
[Emby](https://emby.media/), uno activo a la vez), encuentra carátulas en varios
proveedores y aplica una carátula elegida a tu servidor multimedia, directamente
a través de su API o exportando YAML compatible con
[Kometa](https://kometa.wiki/)/PMM. Se ejecuta como un único contenedor Docker en
un Mac o en un servidor Unraid.

Un asistente guiado de primera instalación te lleva desde elegir un idioma hasta
conectar tu servidor, la clave de TMDB y los proveedores, elegir bibliotecas y una
primera sincronización; y para Plex incluye un inicio de sesión con PIN y el
descubrimiento de conexiones, así que nunca tienes que buscar un token o una URL.

## Qué hace

<CardGrid stagger>
	<Card title="Sincroniza tu biblioteca" icon="seti:db">
		Sincroniza tus bibliotecas de películas y series y resuelve cada título a un id de TMDB para que
		los proveedores de carátulas puedan consultarse con precisión.
	</Card>
	<Card title="Encuentra carátulas" icon="seti:image">
		Despliega la búsqueda entre los proveedores habilitados —MediUX, Fanart.tv, TMDB y ThePosterDB—
		y compara las candidatas de pósters y fondos agrupadas por proveedor y conjunto. Usa un conjunto
		entero, mezcla piezas de varios proveedores o arma un conjunto personalizado a partir de una URL
		pegada o un archivo subido.
	</Card>
	<Card title="Aplica carátulas" icon="approve-check">
		Aplica una carátula elegida mediante la API del servidor multimedia (sube y bloquea el campo, al
		instante) o mediante una exportación de YAML de Kometa que tu instancia de Kometa consume en su
		próxima ejecución.
	</Card>
	<Card title="Mantén la localización" icon="translate">
		La interfaz se ofrece en inglés, español, chino simplificado, japonés y portugués de Brasil, con
		reserva al inglés por clave, gestionada a través de Weblate.
	</Card>
</CardGrid>

El trabajo sobre toda la biblioteca se ejecuta como tareas en segundo plano con
progreso en vivo mediante Server-Sent Events, mostrado directamente en el Panel.
Un registro de **Actividad** granular (Ajustes → Actividad) y un registro de
archivo rotativo anotan cada evento operativo, y un comprobador de actualizaciones
con un modal de "Novedades" da a conocer las nuevas versiones.

## Stack

- **SvelteKit** (TypeScript) sobre **Bun**, compilado con `adapter-node`
- **SQLite + Drizzle ORM** (libsql) para la caché de la biblioteca, las
  candidatas, el historial, las tareas y los ajustes
- **Tailwind CSS v4**, una interfaz oscura centrada en las imágenes
- Una cola de tareas en proceso con **Server-Sent Events** para el progreso en
  vivo
