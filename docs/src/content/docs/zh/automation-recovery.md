PosterPilot 的日常工作持久且审核优先。同步、发现、重试和已确认应用都作为持久任务运行；自动化可以创建审核工作，但不会自动应用海报。

## 持久任务

仪表板会实时显示排队、运行、等待重试和终态任务。刷新或离开页面不会取消任务。等价请求会复用活动任务；重叠修改会被阻止并返回冲突任务引用。

每个任务保存服务器／媒体库范围、不可变输入、尝试记录、结果摘要和已净化的目标失败。重启后安全工作会恢复。中途打断的非幂等海报写入会留给审核，而不是盲目重放。

**取消**只请求停止，不删除已提交结果。部分失败时，**仅重试失败项**只创建符合条件的关联工作。验证、凭据或陈旧计划错误需要修正并重新预览。

## 审核优先自动化

在**设置 → 自动化**中为当前服务器创建自动化，并选择：

- 一个或多个媒体库；
- 间隔、每天本地时间，或事件（新项目／同步完成）；
- IANA 时区；
- **同步**或**同步并发现**；
- 可选的已保存审核视图；
- 补跑窗口和连续失败暂停阈值。

默认动作为 `sync_discover`。每次发生都会冻结输入并创建或复用持久任务。编辑只影响未来发生项。服务在补跑窗口内恢复时只补一次；重复事件会合并。

:::important
自动化只同步并可选地发现候选项，然后放入 Review。它不会创建应用任务。
:::

### Webhook

在自动化中生成凭据。端点和令牌只显示一次。请在 `X-PosterPilot-Webhook-Token` 请求头中发送。轮换会使旧令牌失效；禁用会删除令牌。不要把它放入 URL 或日志。

## 重试前先诊断

**设置 → 诊断**对每个命名服务器、TMDB、海报提供方、Kometa 路径、数据目录和备份存储运行独立、无修改检查。结果区分不可达、凭据缺失／被拒、超时和路径读写问题，并显示每个服务器实例的海报能力。

最新结果与上次成功会跨重启保存。提供方故障时可保留已知候选并标记为陈旧；之后成功的空响应会清除旧候选。

已脱敏支持包只有在明确导出时生成。默认不包含媒体标题；无法证明安全的可选项会被省略并写入清单。

## 应用备份

在**设置 → 备份与恢复**创建备份。PosterPilot 会生成一致的 SQLite 快照，并在数据目录下保存包含校验和、应用／架构版本、密钥模式及外部路径引用的包。不会复制媒体服务器或外部挂载的 Kometa 内容。

使用 `.app-key` 时，生成的密钥会包含在包中。使用 `APP_SECRET` 时，密钥不会被包含，恢复必须提供相同的有效值。

可以验证、导出或删除备份。导出前需再次确认，因为包可能包含凭据和密钥材料。按数量／天数保留只会清理未保护的有效包；手动和恢复前安全备份默认受保护。

## 恢复流程

1. 选择**预览恢复**。
2. 检查校验和、SQLite 完整性、架构／迁移、空间、密钥和外部路径警告。
3. 确认未变化的计划及替换范围。
4. PosterPilot 进入维护模式，阻止新修改，等待活动任务结束，并创建受保护的安全备份。
5. 重启容器；替换会在 libsql 打开之前进行。
6. 查看就绪报告。替换或迁移失败会回滚到安全备份。

外部服务不可用在本地完整时只是警告；校验和、数据库、过新的架构、路径或密钥不兼容会阻止恢复。

:::caution
不要手工替换正在使用的 SQLite 文件。在验证媒体库、凭据、Kometa 路径、自动化和服务器范围前保留安全备份。
:::
