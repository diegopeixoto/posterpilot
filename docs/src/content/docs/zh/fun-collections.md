FUN 与合集复用已同步媒体库。它们不会猜测媒体身份，也不会因为找到结果或视觉系列就应用海报。

## 启用 FUN

设置 `FUN_ENABLED=true` 或在设置中启用 **FUN**。禁用时导航项隐藏，`/fun` 路由返回未找到。筛选仅作用于当前命名服务器，抽取前显示准确的符合数量，也不会悄悄放宽无效区间。

## 今晚看什么

从符合池中返回最多三个不同选择。可按媒体库、类型、流派、年份、观看状态、时长、最低评分、添加时间或预设筛选。URL 保存种子、规范化筛选和项目 ID，因此只要项目仍存在，刷新、返回和分享都会重现相同顺序。

重新抽取保留筛选并使用新种子。有足够替代项时会避开当前会话最近历史；池较小时只放宽重复规避，不放宽筛选。盲选模式在揭晓前隐藏身份，胶囊预设会在抽取前显示规则。

## Poster Match

需要一个至少有两个海报候选的项目。每轮从两张图中选择，直到有限淘汰赛只剩一个。无法加载的候选会尽量移除。获胜者保留来源，只会被**暂存**；应用仍需正常预览和确认。

## 氛围画廊

全屏显示海报、背景或混合模式，并支持前后、暂停／继续、间隔和退出的键盘操作。启用 `prefers-reduced-motion: reduce` 时，自动播放默认暂停，直到本次会话中明确启动。加载失败图片会在当前会话跳过。

## 观影时段规划

选择两部或三部电影及总时长预算。规划器只使用已知正时长的不同电影，遵守媒体库、流派、观看状态和评分筛选，绝不会超出预算。重新规划会保留约束并更换种子。

## 合集与系列电影

合集按服务器隔离，来自提供方原生成员关系或 TMDB `belongs_to_collection`。不同服务器同名合集不会合并，也不会仅凭标题相似建立成员关系。

详情页显示来源、作为上下文的本地缺失 TMDB 成员、当前／暂存海报、提供方／套装／作者／语言／设计系列证据，以及可解释的一致性覆盖。未知来源不会被当成刻意不匹配。

## 协调建议与覆盖

具有可验证证据并覆盖多个成员的系列会按覆盖率和海报评分排序。建议会列出已覆盖／未覆盖成员与槽位，只暂存已覆盖项。每个成员和槽位都可单独替换或清除。没有共同证据时，只提供单项候选，不会伪装成协调套装。

## 应用和撤销合集

在合集里选择仍只是普通暂存。协调写入必须预览冻结的成员、槽位、目的地、选择、当前状态和跳过项。成员关系或选择变化会使计划失效。结果按成员和目的地独立保存。

撤销整个合集操作需要对应修订组和新的撤销预览；成员也可从项目历史单独处理。如果当前界面没有为某项能力提供协调操作，请打开成员并使用标准 Review／应用／撤销控件；暂存本身不会写入。

应用前阅读[安全、验证与撤销](../safety/)，并参阅[使用](../usage/)。
