PosterPilot 可管理多个 Plex、Jellyfin 和 Emby 实例，同时严格隔离媒体库、项目、任务、修订、合集、审核状态和自动化。

## 升级现有安装前

1. 等待所有修改任务完成。
2. 备份完整 `/data`，包括数据库／WAL、海报快照、备份和 `.app-key`。若已有应用备份功能，再创建并验证一个手动备份。
3. 记录当前服务器类型、URL 与凭据来源，并保留相同的 `APP_SECRET` 或 `.app-key`。
4. 拉取新镜像并正常重启。不要创建空数据库或手工执行 SQL 迁移。

## 迁移会做什么

数据库迁移会创建一个受保护的命名实例 **Default server**，并将现有服务器数据分配到稳定的 `legacy-default` 范围。启动时会把有效的旧连接（环境变量仍优先）写入该实例并设为活动服务器。

迁移是事务性且幂等的。现有项目 ID、候选、选择、忽略状态、任务、修订、历史和缓存媒体库都会保留，不需要破坏性重新同步。未配置的新安装不会创建虚假连接，而会继续进入设置向导。

:::note
`SERVER_TYPE`、`PLEX_*`、`JELLYFIN_*`、`EMBY_*` 描述受保护的默认连接。其他命名服务器在设置中创建并保存各自加密凭据；环境变量不会定义任意服务器列表。
:::

## 升级后检查

1. 在**设置 → 服务器**确认类型、URL、凭据标志和活动状态。
2. 测试连接并运行**诊断**。
3. 检查媒体库、Review、合集、仪表板／任务和一个项目时间线的数量。
4. 确认选择的媒体库以及 Kometa 的 Plex 绑定。
5. 运行增量同步；只有需要重新读取所有项目时才使用完整扫描，它会保留修订。

如果迁移或解密失败，请停止新容器并恢复升级前数据卷或使用已验证恢复流程。不要在部分复制的数据库上继续运行。

## 添加与切换服务器

在**设置 → 服务器**输入唯一名称、类型、基础 URL 和可重用凭据，先测试再添加。Plex 使用令牌；Jellyfin／Emby 使用 API 密钥或访问令牌。浏览器不会收到已保存密钥。

启用两个以上实例时，使用外壳切换器或**设为活动**。页面、筛选和 Review 视图保持在所选范围。任务和自动化也绑定命名服务器；不同服务器的独立工作可并行。海报能力由每个实例及版本决定。

## Kometa 绑定

Kometa 仅适用于 Plex。设置 `KOMETA_SERVER_INSTANCE_ID` 或在设置中选择命名 Plex 实例。预览会验证绑定；Jellyfin／Emby 或隐式借用另一 Plex 凭据会被拒绝。

## 跨服务器应用

跨服务器应用始终是明确操作，必须共享精确 TMDB、IMDb 或 TVDB 标识；相似标题不够。预览会列出服务器／项目、能力、槽位、当前状态、选择和跳过项。每个目的地都有独立修订和验证。

普通应用绝不会传播。如果当前界面尚未提供跨服务器选择，预览／确认 API 仅面向受控集成；不要通过在预览和确认之间切换活动服务器来模拟。

## 禁用、断开或清除

- **禁用**阻止新修改，但保留凭据、缓存和历史。
- **断开**删除凭据、禁用自动化并保留范围历史。
- **永久清除**仅在断开后可用，会先显示准确影响和备份提示，再要求独立确认；活动修改任务会阻止它。

迁移的默认服务器标记为 **Legacy**，受保护且不能通过普通编辑／清除流程删除。清除前请创建备份并检查项目、任务、修订、合集、自动化和快照影响。
